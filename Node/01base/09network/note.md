# 一、OSI 七层模型（open system Interconnection）

### 分层的目的：

OSI 七层模型，是理想化的模型，将复杂的流程分解为几个功能，实现复杂问题简单化；

## 分层内容：

- 7.应用层：为应用程序提供服务（用户最终的接口）微信，QQ; HTTP\DNS\TFTP\SMTP\DHCP
- 6.表示层：数据格式化、数据加密(数据的表示、安全、压缩)
- 5.会话层：建立、管理和维护会话
- 4.传输层：建立、管理和维护端到端的连接；TCP 协议、UDP 协议
- 3.网络层：（路由器）IP 选址及路由选择；ipv4,ipv6;
- 2.数据链路层：提供介质访问和链路管理(交换机，网卡)
- 1.物理层：物理层（物理设备、网线、光纤）

## 1.1 IP 地址和 MAC 地址（ip ->（换取）mac 地址，是只支持局域网的）

- 最终通信都是通过 MAC 地址通信，网卡中都有自己的 mac 地址(原则上唯一)
- 通过 DHCP 协议分配给每台电脑 IP，通过 ARP 协议将 IP 地址转换成 mac 地址

## 1.2 路由器和交换机

- 交换机：维护 MAC 地址表(交换机端口对应的 MAC 地址)不关心 IP 地址。核心就是交换数据，交换效率高。
- 路由器:(lan 口、wan 口)，在不连接 wan 口的情况下，路由器可以看成是交换机。

# 五、TCP 协议（传输）

- tcp 传输控制协议 Transimision Control Protocal 可靠、面向连接的协议,传输效率低 (在不可靠的 IP 层上建立可靠的传输层)。 TCP 提供全双工服务，即数据可在同一时间双向传播。数据是无序地在网络间传递，接收方需要有一种算法在接受到数据后恢复原有的顺序。
- 源端口号、目的端口号，指代的是发送方随机端口，目标端对应的端口
- 32 位序列号是用于对数据包进行标记，方便重组（4G）
- 4 位首部长度：单位是字节，4 位最大能表示 15，所以头部长度最大为 60 (一般为 20 个字节)
- URG:紧急新号、ACK:确认信号、PSH:应该从 TCP 缓冲区读走数据、RST：断开重新连接、SYN:建立连接、FIN:表示要断开
- 16 位窗口大小：一次发送多少数据（64k）
- 16 位校验和：用来做差错控制，看传输的报文段是否损坏。
- 16 位紧急指针：用来发送紧急数据使用。
- 一个 TCP 段的头部是 20 个字节， 如果每次传递一个数据光是头部就要 20 字节，因此 tcp 传输的过程中可以做粘包。

TCP 是提供可靠的网络传输，需要建立连接，流速控制。UDP 协议只发送包，无连接协议，速度快。
